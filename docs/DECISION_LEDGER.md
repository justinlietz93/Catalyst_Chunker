# Decision Ledger

This ledger tracks the implementation status of accepted architecture decisions.

| ADR | Decision | Status | Implementation Status |
|---|---|---|---|
| ADR-0001 | Catalyst uses EBA layer admission, not service/repository architecture. | Accepted | In progress |
| ADR-0002 | Source lineage is mandatory for every accepted chunk. | Accepted | In progress |
| ADR-0003 | Fixed-size slicing is fallback only, never default. | Accepted | In progress |
| ADR-0004 | Structure-first paragraph/header grouping is the default prose strategy. | Accepted | In progress |
| ADR-0005 | Code chunking requires AST-aware observation by default. | Accepted | Not started |
| ADR-0006 | Semantic chunking is a refinement strategy, not the primary strategy. | Accepted | Not started |
| ADR-0007 | Retrieval uses parent/child and sentence-window recovery instead of blanket overlap. | Accepted | Not started |
| ADR-0008 | Docling is admitted as a boundary adapter, not as an internal model. | Accepted | Not started |
| ADR-0009 | Public projections require schema versions. | Accepted | In progress |
| ADR-0010 | Rejected candidates remain inspectable. | Accepted | In progress |
